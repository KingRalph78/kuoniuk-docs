1. Kuoni System overview

The Kuoni tech stack is currently stands as the following.

InRiver (Content database) contains the site taxonomy and content both factual and marketing lead. It integrates with Celum (digital asset management) were images are stored and published in the various sizes. These images are published and hosted on a AWS CDN. The url paths are kept in InRiver for the site to reference.

Overnight an XML feed is created known as the catalogue cache. The main website uses this to populate content and structure automatically (destinations specific pages not manual ones). This also feeds into PIXIE were our experts create and manage routes and leads. These saved quotes can be visible with mykuoni.

From kuoni.co.uk you can link into the store finder. Here you search for a store and the experts within it. This is also where out CTA panels are and these are iframed into our pages. The various contact forms are created here and feed PIXIE.

A pricing file comes from Travelbox and is consumed in the N2 main site. Also we link via a widget panel on page into the booking flow which sits on the Travelbox domain. We also currently link to a pandemic support site via a panel on the main site pages.

  • InRiver and Celum are edited by Marketing with IT support.
  • PIXIE and mykuoni is maintained by Infogain with IT support
  • Drupal sites Store finder and covid advice are maintained with O&G
  • Travelbox is maintained by Codegen with IT support
  • Kuoni N2 site is supported by Zolv

2.3 Panda changes to N2 operation

With the Panda project release we effectively manage two sites with in one cms. Only the domain changes within IIS to show witch template is requires.

There are now two start pages (home pages). One for the main site and one for retail. Also for Navigation and footer covered later. Everything is still edited within N2.

On any page within N2 there is now an option to how the page behaves.

These are accessed via the edit menu in N2 in the right hand column “Site Channel”

Options are:

  1. AsParent - Take the behaviour from its parent page.
  2. TourOp - The page is normal kuoni.co.uk domain page
  3. TourOpPaused - Visible on kuoni.co.uk Has templated text about being a paused destination and the page replaces all CTA’s with a paused get in touch form to add to a database to inform people later of new brand.
  4. Retail - Page redirects to kuonistores.co.uk domain CTA’s are all set to retail version.
  5. None - The cache is ignored and the page becomes invisible to the public, but available in N2.

4.2 Offer errors

Quite often this produces errors, the most common would be missing a CMP code. The others would be group id with spaces in. N2 does give a good feedback on the reason.

With a missing code or image N2 will give the line number of the offending data. Copying the product name and paste into the website search. Visit that page then in the edit menu, look at the right hand panel and you will see the Catalogue Item Identifier. This is the code to paste into the relevant column on the spreadsheet. The offer will now load with the correct image.

5. Offer of the week

During a normal year we would have 1 or two “Offers of the week”.

These can be found under /offers tree in N2

We built a new template that makes this easier. In most cases the offer may of existed before and just requires editing. Otherwise it was the old version or brand new and the template will have to be used from scratch.

5.1 Offer of the week template

Information on the offer will come from trading in a word document and a copywriter will contribute also.

The new template is makes it easy to create with these sections.

Every offer should be set to be

  • Exclude page from navigation
  • Exclude page from search engine
  • Exclude page from site search engine
Section 1

Self explanatory here on the offer details.

Section 2

Rich text part copywriter text to be placed here.

Section 3

Using a CMP id (the same as offer spreadsheet) product info can be automatically added. It can also be overridden if need be with the empty fields above. Images that create the hero carousel are chosen here. They are the paths to the main image gallery on the product page. This is manual to give an amount of art direction.

Another product can be added if it's a multi-centre.

Section 4

Inclusions are added in a text field and a icon is chosen to display next to it.

Section 5

Departure details are added into these fields and free text.

Section 6

Airline info is added here with more open text field to accommodate the varying airline offers.

There are a couple of droppable parts containing contact info directly on the page. These are best copied from another offer if a fresh template is used.

Once created the offer link is to be checked and will be communicated when it can be live. Technically it is is live already but hidden from Google and the navigation. Access is only possible by the link. Trading send the link to the dealchecker website and inform the sales staff.

6. Catalogue spots

Catalogue spots are kept on one admin page within N2. This gives a number of regions to add content across destination pages.

It gives options to where these should be displayed.

6.1 CTA’s

Destination CTA’s are here and have been set since the Panda project.

The CTA’s are named with the destination they are intended for and placed in a drop zone area “Below Enquire Now Zone” to be visible in the right hand side of destination pages.

There is also a Retail CTA set for the stores domain and and two forms for paused product. These show when we set a page to either Tour op, retail or paused.

Items added to the catalogue spot page have a choice to be:

  • Catalogue spot (Rich text part)
  • Catalogue spot for PTExpert (are html part)

CTA’s contain a iframe tag with a source pointing to expert finder with a string for destination.

eg. https://expert.kuoni.co.uk/expert-cta?destination=Indian%20Ocean

Each item has a content area where the iframe code is set and a “part setting” where a list of check boxes exist for page types within a destination. Then at the bottom a read to set the destinations the part is to display.




6.2 Covid panels

These panels are set the same way as the CTA’s and have a link to the relevant destination in the covid advice panel. eg.

https://covidtraveladvice.kuoni.co.uk/destination/sri-lanka

7. Reusable parts

Multiple reusable part libraries can be created keeping items like brochures parts contained in one single library.

Reusable parts can be manually dropped onto content areas in pages. In “organise parts” mode on a page drop on a reusable part and select the library you want then add the name of the item. This will now be linked to that one source. Changing the source or the element content will change it in all areas it's been added.

7.1 CTA’s

CTA iframes are replicated as reusable parts that can be dropped on manual pages.

7.2 Brochure ads

Brochure parts are links to the order brochure page as reusable parts. These need updating periodically when new titles are released. Once updated all pages with the part on will update.

7.3 Banners

Many banners are placed throughout the site for example Alfred& banners. Being reusable these can be updated at source or added to new pages.

8. Brochure ordering and FTP

The brochure page and ordering system has brochures set to view online and to request a postal version sent via FTP.

From the edit menu an intro can be written and a link to our postcode service is implemented.

From the Global page settings yo get the following:

Clicking edit on the “Add Multiple Brochure” presents the edit menu fore brochure content.

Heading and description for the latest brochure section are self explanatory. Then numerous groups are set for example Indian Ocean, with or without a description.

  • Brochure code - The code given to identify the brochure with BP and Rcubed.
  • Title - Name of the title.
  • Brochure Group - Drop down of created groups to place the title in.
  • Brochure Date - Validity date if needed.
  • Brochure Description - Short description of the title.
  • Store Text - Not in use any more but used to say pick up in store.
  • New - Adds a new label and places the title in the latest brochures group as well as any other selected.
  • Brochure link - the path to the pdf to view online.
  • Brochure image link - path to the image of the front cover.
  • Brochure types - Check the box to display online and post option.

Clicking edit FTP details will present the current 3 ftp destinations we have.

N2 creates a css file pof requests that is uploaded to these paths set.

The file name looks like this

LIV_MKTG_01072021_000029_AM.csv

Run job option turns it on or off, if set to LIV for live the file will start LIV_

All FTP details are to remain as they are.

N2 jobs are are the recurring scripts that are performed by N2 to consume or send data. They consist of jobs that consume the catalogue cache and sending brochure data to FTP.

If one of these jobs is red, there is an error and you can read the error message to inform Zolv.

22.2 Development Design & briefing / retainer

We have a 500 hour retainer with Zolv to develop with (all burnt on Panda project currently) to develop on N2, the process is as follows:

  1. Zolv create a work package folder in Target process
  2. Kuoni follow a design and prototype process
  3. Design is handed over through TP with detailed written spec
  4. Other developments are also added into the work package
  5. A release process is is carried out.

Releases have been planned for 5 times a year or more recently a more project ad hoc basis (this requires Zolv resource to be available).

22.3 Release management

During a release a process must be followed as to not lose any work in-between switching servers with our IT partners in the Nordics:

  • Kuoni enter content freeze
  • Nordics patch and content sync STG
  • Zolv deploy release into STG (plus run any N2 jobs or data initialisation (SQL scripts etc))
  • Kuoni test STG and confirm ok
  • Nordics switch STG servers into live environment
  • Kuoni exits content freeze
  • After agreed amount of time ie 1 day or 1 week
  • Kuoni enters content freeze
  • Nordics patch and content sync LIV from STG
  • Nordics switch LIV servers back into live environment
  • Nordics shut down STG

For support at Zolv Richard Pickles is our contact richard@zolv.com
